## Overview

This project is a web application built with Flask that simulates an online clothing store. It includes both admin and customer routes that directs the user to different functions they can do. It allows users to do things such as browse inventory and create a wishlist.

## Database Design

### SQL (MySQL on RDS)

- `[clothing]` - stores [clothing_id, name, category, base_price]; primary key is `[clothing_id]`
- `[attributes]` - stores [attribute_id, color, material]; primary key is `[attribute_id]`
- `[clothing_attribute]` - stores [clothing_id, attribute_id, stock_qty, sku]; primary key is `[id]`

The JOIN query used in this project: For each SKU in the clothing_attributes table, look up the clothing item it belongs to, and also look up the attribute (like color) associated with it. Then combine all that information into one row.
CODE:
FROM clothing_attributes ca
JOIN clothing c ON ca.clothing_id = c.clothing_id
JOIN attributes a ON ca.attribute_id = a.attribute_id

### DynamoDB

- **Table name:** `[Wishlist]`
- **Partition key:** `[wishlist_id]`
- **Attributes:** `[wishlist_id, name, sku, color, price]`
- **Used for:** [Used to manage a customer wishlist. Do things like view wishlist, add to wishlist, update item in wishlist, and clear wishlist. This is a way to store customer-specific data in a similar format to the MySQL inventory table.]

## CRUD Operations
# MySQL Inventory

| Operation | Route            | Description                                     |
| --------- | ---------------- | ----------------------------------------------- |
| Create    | `/add-item`      | [Adds a new clothing item to the MySQL database]|
| Read      | `/browsetextbox` | [Lets the user search for items by category]    |

| Delete    | `/delete-item`   | [Deletes an item from the inventory using SKU.] |

| Operation | Route              | Description                                               |
| --------- | ------------------ | --------------------------------------------------------- |
| Create    | `/add-to-wishlist` | [Inserts a new wishlist item into wishlist]               |
| Read      | `/view_wishlist`   | [Retrieves and displays all wishlist items]               |
| Update    | `/update-wishlist` | [Updates an existing wishlist entry based on wishlist_id] |
| Delete    | `/clear-wishlist`  | [Deletes all wishlist items]                              |

## AI Assistance

I used Claude to help me with some things including:
- Generating tables for my MySQL database: "Write a MySQL schema for a clothing shopping app. I need tables for clothing, colors/materials, and a key linking the tables so we can join them. Include 5-10 sample INSERT statements per table"
- Instead of a textbox, I wanted to use a dropdown menu where users can select one of the categories (when adding an item into the MySQL table, or when searching for items in a specific category). I used AI to figure out how to adjust my code to do this: Added [categories = ["tops", "bottoms", "outerwear", "other"]] into my add-item and browse routes.
- I was having a lot of troubles figuring out how to apply an added item to all 3 of my database tables correctly. Eventually, AI suggested just creating a function execute_insert so each INSERT/UPDATE/DELETE step happened cleanly and consistently. This is used in my add-item, delete-item, and update-price routes.
- I wanted each added wistlist item to have it's own unique code. I used AI to find a package allowing me to do this. It recommended using the uuid package, which I used within my add-to-wishlist route: ['wishlist_id': str(uuid.uuid4()),]
- I was having trouble getting my dynamoDB functions to work (delete_item, update_item, put_item) for my wishlist routes. I used AI to help me get the syntax to write this code.
